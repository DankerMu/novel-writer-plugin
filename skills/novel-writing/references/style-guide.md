# 去 AI 化规则详解

本文档定义 novel 插件系统的完整去 AI 化策略，供 ChapterWriter、QualityJudge 参考。

## Layer 1: 风格锚定（输入层）

### 风格指纹提取

WorldBuilder（风格提取模式）从用户样本中提取以下可量化特征：

- **avg_sentence_length**：平均句长（字数），用于校准生成文本的句式节奏
- **dialogue_ratio**：对话占全文比例，控制叙述与对话的平衡
- **rhetoric_preferences**：修辞偏好列表（频率标注），如比喻、排比、短句切换
- **forbidden_words**：作者从不使用的词汇（精准收录，不过度泛化）
- **character_speech_patterns**：角色语癖，需有样本中的具体例句支撑
- **writing_directives**：正向写作指令（可执行的风格指南数组），用于直接注入 ChapterWriter prompt

### 风格样本降级方案

当用户无自有样本时：
1. **仿写模式**（推荐）：指定参考作者 → 提取风格指纹 → 标记 `source_type: "reference"`
2. **先写后提**：用户先写 1 章 → 再提取
3. **预置模板**：选择预设风格模板（轻松幽默/热血少年/细腻言情等）→ 微调

### 风格漂移监控

每 5 章提取一次当前输出的风格特征，与 style-profile.json 对比：
- 句长偏移 > 20% → 警告
- 对话比例偏移 > 15% → 警告
- 新出现的高频 AI 用语 → 追加到黑名单

**自动纠偏**：当漂移超阈值时，将偏离项写入 `style-drift.json`，自动注入后续 ChapterWriter 的 context：
```json
{
  "detected_chapter": 25,
  "drifts": [
    {"metric": "avg_sentence_length", "baseline": 18, "current": 24, "directive": "句子过长，回归短句节奏"},
    {"metric": "dialogue_ratio", "baseline": 0.4, "current": 0.28, "directive": "对话偏少，增加角色互动"}
  ],
  "injected_to": ["ChapterWriter"]
}
```
纠偏指令会持续注入直至下次检测确认回归基线（偏移 < 10%）。

## Layer 2: 约束注入（生成层）

ChapterWriter prompt 中注入以下硬约束：

### 2.1 AI 用语黑名单

从 `ai-blacklist.json` 加载，生成时完全禁止。包含但不限于：
- 情感描写类：不禁、莫名、油然而生、心中暗道、嘴角微微上扬
- 过渡连接类：与此同时、值得一提的是、毫无疑问
- 形容夸张类：宛如、恍若、仿佛置身于
- 详见 `${CLAUDE_PLUGIN_ROOT}/templates/ai-blacklist.json`

### 2.2 角色语癖

主角和核心角色可定义口头禅或说话习惯（可选，非强制）：
- 口头禅出现频率：偶尔点缀即可（每 3-5 章出现一次为宜），一章出现一次都嫌多
- 语癖需符合角色背景（文化人用文言、江湖人用俚语）
- 角色区分优先靠说话风格（正式/随意、长句/短句）、用词层次、性格驱动的表达方式，而非口头禅

### 2.3 反直觉细节

在场景允许时融入"反直觉"的生活化细节（属于 §2.9 工具箱的 sensory_intrusion / fragment_detail 技法），例如：
- 打斗中途想起锅里还炖着汤
- 修炼突破时被蚊子咬了一口
- 严肃对话中对方裤子上有个洞

目的：打破 AI 生成文本的"完美感"和"刻板感"。

### 2.4 场景描写限制

- 场景描写最多 2 句，优先用人物动作带出环境
- 禁止大段环境白描（"空气中弥漫着……远处是……近处有……"）
- 好的范例：`他一脚踢开歪斜的门板，霉味扑面——这地方至少荒了三年。`

### 2.5 句式多样性

- 禁止连续 3 句相同句式（如连续 3 个"他……"开头）
- 长短句交替：2-3 个短句后接 1 个长句，或反之
- 避免排比过度（连续排比 ≤ 3 项）

### 2.6 标点符号约束

LLM 对特定标点有系统性过度使用倾向，是最容易被读者感知的 AI 痕迹之一：

- **破折号（——）限频**：每千字 ≤ 1 处。LLM 极度偏好用破折号做解释性插入、情绪停顿和场景切换，频率远高于人类写手。应改用逗号、句号或重组句式
  - ✗ `他看向远方——那是他曾经的家。` → ✓ `他看向远方。那是他曾经的家。`
  - ✗ `门开了——是她。` → ✓ `门开了，是她。`
  - ✗ `一切都结束了——或许吧。` → ✓ `一切都结束了。或许吧。`
- **省略号（……）限频**：每千字 ≤ 2 处，禁止连续两句以省略号结尾
- **感叹号（！）限频**：每千字 ≤ 3 处（对话内不计），叙述文中避免连续感叹

### 2.7 对话与内心活动格式

人物说话和内心活动统一使用中文双引号（""），格式如下：

- **说话**：`XX说："我出去了。"` 或 `"我出去了。"XX甩手离开。`
- **内心活动**：`XX心想："关我什么事。"` 或 `XX暗道："这人不对劲。"`
- **禁止**使用单引号（''）、直角引号（「」）、英文引号（""）包裹对话和心理活动
- 引号内的句末标点在引号内：`"走吧。"` 而非 `"走吧"。`

### 2.8 统计分布目标（反困惑度/反均匀度检测）

AI 生成文本在统计分布上呈现异常均匀性，是区别于人类写作的核心特征。以下 6 个维度供 ChapterWriter 作为内隐参照，不作为逐项硬门槛：

| 维度 | 人类范围 | AI 特征 | 策略 |
|------|---------|--------|------|
| 句长方差 | std_dev 8-18 | < 6 | 穿插极短句(2-5字)和长句(35+字)，节奏跟情绪走 |
| 段落长度变异 | CV 0.4-1.2 | < 0.3 | 允许单句段落和长段内心独白，不追求均匀 |
| 词汇多样性 | 500字内同一修饰词不复现 | 高频复用"缓缓""微微" | 动作替代形容词，偶尔用低频词 |
| 叙述连接词 | 叙述中极少使用 | 频繁"然而""因此" | narration_only 词在叙述中零容忍 |
| 语域混合 | 同章自然波动 | 全文统一腔调 | 紧张→口语化，回忆→文学化 |
| 情感弧线 | 有突变和矛盾 | 单弧线平滑推进 | 让情感跟着角色处境自然波动，不强制频率 |

### 2.9 人性化技法工具箱（随机采样）

以下 12 种技法供 ChapterWriter 按场景需要随机采样，不要求每章全覆盖：

| 技法 | Tag | 说明 |
|------|-----|------|
| 思维中断 | `thought_interrupt` | 角色思考被外部事件打断，思路断裂不回收 |
| 感官突入 | `sensory_intrusion` | 不相关的感官细节突然侵入叙述（远处狗叫、衣领扎脖子） |
| 自我纠正 | `self_correction` | 叙述者或角色说了一半改口，或内心推翻刚才的判断 |
| 未完成思维 | `incomplete_thought` | 一个念头起了个头就被打断或遗忘，不刻意补完 |
| 语域突变 | `register_shift` | 文风在同一段内发生跳变（正式→粗口、文言→白话） |
| 言行矛盾 | `say_do_mismatch` | 角色嘴上说一套，身体/行为做另一套 |
| 身体先于思维 | `body_before_mind` | 先写身体反应（拳头攥紧、后退一步），再补意识层面的理解 |
| 碎片信息 | `fragment_detail` | 给出零碎的、未经整合的观察细节，不做总结性陈述 |
| 情感急转 | `emotion_whiplash` | 情绪在极短篇幅内（2-3 句）剧烈翻转，不做平滑过渡。与 §2.8 情感弧线的区别：§2.8 关注全章弧线形状（是否有突变节点，可由情节自然驱动），emotion_whiplash 是局部段落技法 |
| 节奏突变 | `rhythm_break` | 长段叙述后突然一个两三字短句，或反之 |
| 非线性叙述 | `micro_nonlinear` | 在段落级别打乱时序（先写结果再补原因，闪回一个细节） |
| 不完美对话 | `imperfect_dialogue` | 对话中有打岔、重复、词不达意、答非所问 |

> **使用原则**：让技法从场景中自然生长。某章没有适合场景就不用——这种"不用"本身就是随机性。
> QualityJudge 不检查单章计数，仅记录 tag 供 dashboard 跨章统计。连续 5 章零技法时输出 risk_flag `humanize_drought`。

## Layer 3: 后处理（ChapterWriter Phase 2）

### 润色规则

ChapterWriter Phase 2 对初稿逐项执行：

1. **黑名单扫描**：全文搜索 `ai-blacklist.json` 中所有词条
2. **逐个替换**：命中项替换为风格相符的自然表达，替代词需符合上下文语境
3. **标点频率修正**：
   - 破折号（——）超过 1 处/千字 → 逐个替换为逗号、句号或重组句式
   - 省略号（……）超过 2 处/千字 → 削减或改用具体描写
4. **句式调整**：
   - 句长偏离 style-profile 的 avg_sentence_length > 30% 的句子进行拆分或合并
   - 相邻 5 句中出现 ≥ 2 个相同句式 → 改写其中 1 句
4. **修改量控制**：总修改量 ≤ 原文 15%（字数变化比）

### 不可修改项

- 角色对话中的语癖和口头禅
- 情节因果链中的关键句
- 伏笔暗示语句
- 角色名、地名、术语

## Layer 4: 检测度量（QualityJudge）

### 风格自然度评分标准（7 指标范围判定）

| 指标 | 人类范围(4-5分) | 过渡区(3分) | AI特征区(1-2分) |
|------|---------------|------------|----------------|
| 黑名单命中率(次/千字) | 0-2 | 3-4 | ≥ 5 |
| 句式重复率(相邻5句) | ≤ 1 组 | 2 组 | ≥ 3 组 |
| 句长标准差(字) | 8-18 | 6-8 | < 6 |
| 段落长度 CV | 0.4-1.2 | 0.3-0.4 | < 0.3 |
| 叙述连接词密度(次/千字) | 0 | 1-2 | ≥ 3 |
| 修饰词重复(500字窗口) | 0-1 次 | 2 次 | ≥ 3 次 |
| style-profile 综合匹配 | 均匹配 | 部分偏移 | 明显不匹配 |

评分：≥6项人类范围→5分, ≥5项无AI区→4分, ≥4项人类或过渡→3分, 2-3项AI区→2分, ≥4项AI区→1分。

向后兼容：缺失 ≥ 3 项指标时退化为旧版 4 指标评分（黑名单命中率 + 句式重复率 + 破折号频率 + style-profile 匹配度）。

### 黑名单维护机制

- **初始化**：`${CLAUDE_PLUGIN_ROOT}/templates/ai-blacklist.json` 提供 ≥ 30 个常见 AI 高频中文用语
- **持续更新**：QualityJudge 检测到新高频 AI 用语时，建议追加到黑名单
- **用户自定义**：用户可手动添加/删除
- **误伤保护**：如果某个黑名单词是用户风格样本中的高频词，自动豁免
