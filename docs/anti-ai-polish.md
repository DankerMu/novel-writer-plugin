# 反AI味润色指南

> 写作完成后、提交前的最后一道工序。目标：让文字读起来像人写的，不像AI生成的。
> 本文件在 write-chapter 硬校验通过后作为润色检查参考。
>
> **类型适配说明**：本文件的默认值适用于大多数网文平台（以番茄为基线）。如果你的小说是其他类型（科幻/悬疑/恐怖/言情），
> 请查看 `concept.md` 的"类型覆写"节——覆写值优先于本文件的默认值。
> 下文中标注 `⚙️ 可覆写` 的规则支持按类型调整。

---

## 一、AI高频词黑名单（10类 200+词）

> 规则：正文中出现以下词汇，必须替换为更具体、更口语化的表达。
> 不是绝对禁用——对话中角色因身份使用是允许的（如学者角色说"显而易见"）。

### 1. 总结概括词

| 黑名单词 | 替换方向 |
|---------|----------|
| 总而言之、综上所述、总的来说 | 删除，直接说结论 |
| 不言而喻、毋庸置疑、毫无疑问 | 删除，或用动作/反应代替 |
| 归根结底、说到底、本质上 | 用具体事件说明 |
| 一言以蔽之、简而言之 | 删除 |
| 值得注意的是、需要指出的是 | 删除，直接叙述 |

### 2. 枚举模板词

| 黑名单词 | 替换方向 |
|---------|----------|
| 首先…其次…最后… | 用场景转换或动作串联 |
| 一方面…另一方面… | 用对比场景代替 |
| 第一…第二…第三… | 删除编号，融入叙事 |
| 不仅…而且…更… | 简化，保留最有力的一个 |
| 既…又…还… | 拆成独立句 |

### 3. 学术腔/书面腔

| 黑名单词 | 替换方向 |
|---------|----------|
| 显而易见、不可否认 | 删除 |
| 从某种程度上说、在某种意义上 | 删除 |
| 具有重要意义、意义深远 | 用具体影响代替 |
| 基于、鉴于、考虑到 | 用"因为"或直接省略 |
| 诸如、例如（频繁使用时） | 直接给出例子 |
| 旨在、致力于、着力 | 用口语化动词 |
| 深刻地、深深地（修饰抽象词时） | 用具体反应代替 |
| 充分地、完全地、彻底地 | 多数情况可删除 |
| 相应的、相关的、对应的 | 能省就省 |
| 进而、从而、继而 | 用"然后"或直接连接 |

### 4. 逻辑连接词（过度使用时）

| 黑名单词 | 替换方向 |
|---------|----------|
| 然而、但是、不过（连续出现） | 保留一个，删除其余 |
| 因此、所以、于是（密集出现） | 用动作或场景转换代替 |
| 尽管如此、即便如此 | 简化为"但" |
| 与此同时、在此期间 | 用场景切换代替 |
| 换言之、也就是说 | 删除，读者不傻 |
| 事实上、实际上（口头禅式） | 大部分可删除 |

### 5. 情绪直述词

| 黑名单词 | 替换方向 |
|---------|----------|
| 感到震惊、感到愤怒、感到悲伤 | 用身体反应代替：握拳、瞳孔收缩、声音发颤 |
| 内心充满了XX | 用行为展示 |
| 心中涌起一股XX | 用生理反应代替 |
| 不禁感到、不由得 | 删除"不禁"，直接写反应 |
| 一种XX的感觉油然而生 | 用具体感受代替 |
| 百感交集、五味杂陈 | 写出具体是哪几种感受 |
| 心如刀割、心如死灰 | 除非是对话，否则用具体描写 |

### 6. 动作套话

| 黑名单词 | 替换方向 |
|---------|----------|
| 缓缓开口、缓缓说道 | "说"/"问" + 具体动作 |
| 微微一笑、淡淡一笑、轻轻一笑 | 写清楚笑的方式和原因 |
| 眉头微皱、眉头紧锁 | 偶尔用可以，不能反复出现 |
| 深吸一口气 | 每章最多1次 |
| 目光如炬、目光灼灼 | 用具体眼神描写 |
| 嘴角微微上扬 | 同"微微一笑"，换个写法 |
| 脚步一顿、身形一滞 | 每章最多各1次 |
| 不由自主地 | 删除，直接写动作 |

### 7. 环境套话

| 黑名单词 | 替换方向 |
|---------|----------|
| 月光如水、月光倾泻 | 写这个场景里月光的具体样子 |
| 阳光明媚、阳光灿烂 | 用光影细节代替 |
| 微风拂面、清风徐来 | 风带来了什么气味/声音？ |
| 万籁俱寂、鸦雀无声 | 写安静中能听到的小声音 |
| 如诗如画、美不胜收 | 删除，用画面感描写 |
| 璀璨、瑰丽、绚烂、绮丽 | 用具体颜色/形状/光线 |
| 宛如、仿若、恰似（密集使用时） | 每段最多1个比喻 |

### 8. 叙事填充词

| 黑名单词 | 替换方向 |
|---------|----------|
| 就这样、于是乎 | 删除 |
| 话说回来、言归正传 | 删除，叙事不需要"回到正题" |
| 不得不说、不得不承认 | 删除 |
| 说起来、要知道 | 多数情况可删除 |
| 紧接着、随即（密集使用） | 保留一个 |
| 只见、但见 | 每章最多2次 |

### 9. 抽象空词 `⚙️ 科幻类有灰度`

> 科幻中描写超出人类认知的场景时，"难以形容""不可名状"可有限度使用（每章≤2处），但仍应尽量给出具体感官描写。

| 黑名单词 | 替换方向 |
|---------|----------|
| 某种程度上、某种意义上 | 删除 |
| 难以形容、无法言喻、不可名状 | 努力去形容！这是作者的工作 |
| 一种说不清道不明的 | 写出能说清的部分 |
| 极其、非常、十分（修饰抽象词） | 用具体程度代替 |
| 各种各样、形形色色 | 举2-3个具体例子 |

### 10. 机械开合词

| 黑名单词 | 替换方向 |
|---------|----------|
| 时间来到了XX | 用场景/动作切入 |
| 让我们把目光转向 | 直接切换场景 |
| 回到XX这边 | 直接写 |
| 夜幕降临 / 晨光熹微（作为开头） | 用角色动作开头 |
| XX的故事还要从XX说起 | 直接从事件切入 |

## 二、7层结构规则

### 第1层：反模板句式

**禁止**：
- "首先…其次…最后…"三段式总结
- "一方面…另一方面…"二元对立
- "不仅…而且…更…"递进堆砌
- 每段开头用相同句式（排比陈述）

**替代**：用场景推进、对话推进、动作推进，不要用逻辑推进。

### 第2层：形容词/副词密度控制 + 比喻密度控制

| 规则 | 限制 |
|------|------|
| 每300字中强调词（极其/非常/十分/无比） | ≤2个 |
| 每300字中形容词总量 | ≤6个 |
| 连续两个以上形容词修饰同一名词 | 禁止（"美丽优雅温柔的女子"→保留最有力的1个） |
| "的"字连用（"XX的XX的XX"） | 最多2个"的"连用 |
| **每段比喻数量（所有形式合计）** | **≤1个**（精确词条如"宛如" + 通用结构如"像X一样"均计入） |
| **每千字比喻总量** | **≤3个** |
| **通用比喻优先替换** | 替换为本书专属意象（参考 style-profile 意象库） |

### 第3层：四字成语/词组密度控制

| 规则 | 限制 |
|------|------|
| 每500字中四字成语 | ≤3个 |
| 连续使用2个以上四字词组 | 禁止（"心潮澎湃、热血沸腾、激动万分"→留1个+具体描写） |
| 同一段落中四字词组 | ≤2个 |

**特别注意**：四字词组连用是AI写作最明显的特征之一。

### 第4层：对话去AI化

每句对话必须有**意图**，对照检查：

| 意图类型 | 示例 |
|---------|------|
| 试探 | "你今天好像心情不太好？"（不是真问心情，是在试探原因） |
| 回避 | "这事以后再说吧。"（不想正面回答） |
| 施压 | "你确定？再想想？"（迫使对方让步） |
| 诱导 | "如果是我的话，我会选择……"（引导对方做出特定选择） |
| 挑衅 | "就你？"（故意激怒） |
| 敷衍 | "嗯，好。"（表面配合，内心不认同） |

**禁止**：
- 对话中使用"我认为""我觉得我们应该"等书面表达（除非角色身份要求）
- 对话内容与叙述重复（刚描写完一个场景，角色又把同一件事口述一遍）
- 所有角色说话语气相同（检查：去掉对话标签，能否分辨是谁在说？）

**角色声音区分规则**：
- 每个有名字的角色应有至少 1 项可辨识的语言特征（用词层次/句式长度/语气词偏好/方言/口头禅）
- 去掉"XX说"后，仅凭对话内容应能以 ≥70% 准确率判断说话者
- 对话轮数 ≥3 轮时，至少 1 轮应体现角色独有的说话方式

### 第5层：段落结构 `⚙️ 可覆写`

| 规则 | 默认值 | 类型覆写参考 |
|------|-------|-------------|
| 单句段落占比 | 25-45% | 科幻15-30% / 悬疑20-35% / 恐怖30-50% |
| 每段字数范围 | 20-100字（超过80字建议拆分） | 科幻可到120字 / 悬疑可到100字 |
| 连续3段以上相同句式 | 禁止 | 通用 |
| 连续3段以上相似长度（±10字） | 需要调整，制造长短交错 | 通用 |

**节奏口诀**：长-短-短-长-短，如呼吸般起伏。

### 第6层：标点节奏 `⚙️ 可覆写`

| 规则 | 默认值 | 类型覆写参考 |
|------|-------|-------------|
| 省略号（……） | 每段最多1个，每章最多5个 | 悬疑/恐怖可到≤8个 |
| 感叹号（！） | 每段最多1个，每章最多8个 | 科幻收紧到≤5个 |
| 省略号+感叹号同时出现 | 禁止 | 通用 |
| 破折号（——） | **绝对零容忍**，一律替换为逗号/句号/重组句式 | 通用 |
| 问号连用（？？） | 禁止 | 通用 |
| 感叹号连用（！！） | 禁止 | 通用 |

### 第7层：AI 句式原型检测

> 以下 5 类句式原型是 LLM 生成文本的结构性特征，词汇级黑名单无法覆盖。由 CW 在写作/润色阶段做语义自检，QJ 在评估阶段做语义匹配计数。

| 原型 | 判定标准 | 替换方向 |
|------|---------|---------|
| **作者代理理解**（author_proxy_comprehension） | 用全知视角替角色做理解总结，如"她知道他是在……""他明白这意味着……" | 改为角色直接反应（动作/微表情/内心独白），不做认知总结 |
| **模板化转折**（template_transition） | 固定句式制造转折，如"听上去…，可…""看似…，实则…""不是…，而是…" | 用动作/事件制造转折，不用逻辑连接词 |
| **抽象化判断**（abstract_judgment） | 用概念词替代具体描写，如"那股安排劲""可被执行的路" | 拆解为具体感官细节或角色行为 |
| **书面腔入侵**（formal_tone_intrusion） | 公文腔入侵叙事，如"可被执行""具有一定的合理性" | 改为口语化表达或直接删除 |
| **否定-肯定伪深度**（negation_affirmation） | 用"不是A，而是B"制造虚假洞察感，如"不是恐惧，而是某种说不清的期待""不是不在乎，而是太在乎" | 直接写B的具体表现（动作/感官/细节），砍掉否定前缀 |

**豁免**：第一人称视角中"我知道他在……"类自我理解是自然表达，不计入。

**LLM 语义匹配说明**：这 5 类原型的表现形式无限多，无法穷举精确字符串。CW 和 QJ 在 LLM 推理层做语义识别，`ai-blacklist.json` 中的 `ai_sentence_pattern` 类别（`semantic_match: true`）提供原型定义和示例作为参照锚点。lint 脚本跳过 `semantic_match` 类别。"不是X，而是Y"句式同时命中 template_transition 和 negation_affirmation 双重检测，计数时只计 1 次。

## 三、润色执行流程

> 在 write-chapter 硬校验（字数+段落+对话比例+禁用清单）通过后执行。

### Step 0（前置清洗，无条件执行）

1. **模型 artifact 清除**：扫描正文中是否存在 `<thinking>`、`</thinking>`、`<reflection>` 等 LLM 内部标签。发现即删除——这些是模型推理残留，绝不允许出现在小说正文中
2. **引号格式统一**：将所有英文直引号（`"`）、英文弯引号（`""`）、单引号（`''`）、直角引号（`「」`）统一替换为中文双引号（""）

### Step 1：黑名单扫描

通读正文，对照第一部分10类黑名单，标记所有命中词汇。

**判断标准**：
- 该词是角色对话中、因角色身份合理使用的？→ 保留
- 该词在本章中首次出现且不可替代？→ 可保留
- 其他情况 → 替换

### Step 2：结构规则检查

按第二部分7层规则逐项检查：
1. 有无模板句式？
2. 形容词/副词是否超标？比喻是否超密度？
3. 四字词组是否连用/超密度？
4. 对话是否都有意图？是否去掉标签能区分角色？角色声音是否有区分度？
5. 段落长短是否交错？
6. 标点是否过度使用？破折号是否为零？
7. 有无 AI 句式原型（作者代理理解/模板化转折/抽象判断/书面腔入侵/否定-肯定伪深度）？

### Step 3：抽象→具体转换

重点检查：
- "感到XX" → 替换为身体反应/行为
- "非常/极其" → 替换为具体程度
- "难以形容" → 努力形容出来
- 通用比喻 → 替换为本书专属意象

### Step 4：节奏朗读测试

默读全章，检查：
- 有没有连续3句以上节奏相同？（同样长度、同样句式）
- 有没有读起来"卡顿"的地方？（通常是逻辑连接词堆砌）
- 有没有读起来"无聊"的地方？（通常是描写过长、缺少变化）

## 四、快速检查清单

> 时间有限时，至少检查以下5项：

- [ ] **四字词组**：有无连续2个以上四字词组？→ 必须拆开
- [ ] **情绪直述**：有无"感到XX""心中涌起XX"？→ 替换为身体反应
- [ ] **微微系列**：有无"微微一笑""微微皱眉""微微点头"？→ 换具体描写
- [ ] **缓缓系列**：有无"缓缓开口""缓缓说道"？→ 简化或加具体动作
- [ ] **标点过度**：省略号和感叹号各有多少？→ 控制在限额内
- [ ] **破折号零容忍**：正文中有无"——"？→ 一律替换为逗号/句号/重组句式
- [ ] **AI 句式原型**：有无作者替角色做理解总结、模板化转折、抽象判断、书面腔入侵、否定-肯定伪深度（"不是X，而是Y"）？→ 逐个替换
- [ ] **模型标签泄漏**：正文中有无 `<thinking>`、`</thinking>` 等 XML 标签？→ 无条件删除
- [ ] **英文引号**：有无英文直引号 `"` 或弯引号 `""`？→ 替换为中文双引号 `""`
